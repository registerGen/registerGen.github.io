CF52C Circular RMQ

https://www.luogu.com.cn/problem/CF52C

Solution

线段树傻逼题。

首先强行把下标变成从 111 开始。

如果 l≤rl\le rl≤r，就是基本的线段树。

如果 l>rl>rl>r，则在 [l,n][l,n][l,n] 上搞一搞，再在 [1,r][1,r][1,r] 上搞一搞就行了。

唯一有点 tricky 的地方就是判断是哪个操作（

在输进去 222 个数后，我们 getchar 一下，如果是空格，就是修改，否则就是 RMQ。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF413E Maze 2D

https://www.luogu.com.cn/problem/CF413E

翻译是我给的（逃

Solution

线段树。

这题 的简化版。

线段树内维护四个值：每段区间左上到右上，左上到右下，左下到右上，左下到右下的最短路：

struct Node
{
	int uu,ud,du,dd;
};

画个图就能得出 pushUp：

#define CN const Node

inline Node pushUp(CN& x,CN& y)
{
	Node res;
	res.uu=std::min(inf,std::min(x.uu+y.uu,x.ud+y.du)+1);
	res.ud=std::min(inf,std::min(x.uu+y.ud,x.ud+y.dd)+1);
	res.du=std::min(inf,std::min(x.dd+y.du,x.du+y.uu)+1);
	res.dd=std::min(inf,std::min(x.dd+y.dd,x.du+y.ud)+1);
	return res;
}

查询的时候分类讨论判一判就行了。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF1045G AI robots

https://www.luogu.com.cn/problem/CF1045G

Solution

CDQ 分治。

首先按视野从小到大排序，这样只要右边能看见左边就可以保证互相看见。

CF474F Ant colony

https://www.luogu.com.cn/problem/CF474F

Solution

线段树。

稍微分析一下可知，本题就是求 [l,r][l,r][l,r] 中 不等于 gcd⁡i=lrai\displaystyle \gcd_{i=l}^r a_ii=lgcdr​ai​ 的数的个数。

区间 gcd⁡\gcdgcd 和区间等于 gcd⁡i=lrai\displaystyle \gcd_{i=l}^r a_ii=lgcdr​ai​ 个数都有可加性，直接线段树就好了。

inline void pushUp(int i)
{
	t[i].gcd=gcd(t[ls(i)].gcd,t[rs(i)].gcd);
	t[i].sum=(t[i].gcd==t[ls(i)].gcd?t[ls(i)].sum:0)+(t[i].gcd==t[rs(i)].gcd?t[rs(i)].sum:0);
}

CF689D Friends and Subsequences

https://www.luogu.com.cn/problem/CF689D

Solution

二分 + ST 表。

考虑当 lll 固定时，如何快速计算对答案的贡献。

我们发现，当 lll 固定时，f(r)=max⁡i=lraif(r)=\displaystyle\max_{i=l}^r a_if(r)=i=lmaxr​ai​ 单调不减，g(r)=min⁡i=lraig(r)=\displaystyle\min_{i=l}^r a_ig(r)=i=lminr​ai​ 单调不增。

蓝色段的长度就是对答案的贡献。

我们可以二分求出蓝色段的左端点和右端点，求 min⁡\minmin，max⁡\maxmax 时用 ST 表，因为不带修。时间复杂度 O(log⁡n)O(\log n)O(logn)。

总时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF103D Time to Raid Cowavans

https://www.luogu.com.cn/problem/CF103D

Solution

一道很不错的分块思想的题，强烈建议大家去做。

设置一个参数 BBB。

若公差 k≥Bk\ge Bk≥B，则暴力算，O(nB)O\left(\dfrac n B\right)O(Bn​)。

否则，对于 kkk 开一个后缀和数组 sumsumsum，sumi=sumi+k+aisum_i=sum_{i+k}+a_isumi​=sumi+k​+ai​，处理它的复杂度为 O(n)O(n)O(n)。

可以预处理所有小于 BBB 的 kkk 的 sumsumsum，总复杂度 O(nB+n⋅nB)≥O(nn)O\left(nB+n\cdot \dfrac n B\right)\ge O\left(n\sqrt n\right)O(nB+n⋅Bn​)≥O(nn​)，当 B=nB=\sqrt nB=n​ 时取到。

考虑如何优化空间。

把询问离线下来，并把询问按照 kkk 从大到小排序，每次若 k≥Bk\ge Bk≥B 暴力，否则若 kkk 和上一次的不一样则更新 sumsumsum。

由于询问中最多 B−1B-1B−1 个不同的数 <B<B<B，所以复杂度 O(nn)O(n\sqrt n)O(nn​)。

总复杂度 O(nlog⁡n+nn)O(n\log n+n\sqrt n)O(nlogn+nn​)。

CF1093G Multidimensional Queries

https://www.luogu.com.cn/problem/CF1093G

Solution

线段树 + 一个 trick。

绝对值怎么办呢？

由于 kkk 特别小，所以这些绝对值就是 2k2^k2k 个式子取 max⁡\maxmax。例如：

∣x1−x2∣+∣y1−y2∣=max⁡{x1−y1+x2−y2,x1−y1+y2−x2,y1−x1+x2−y2,y1−x1+y2−x2}\begin{aligned}|x_1-x_2|+|y_1-y_2|=\max\{&x_1-y_1+x_2-y_2,x_1-y_1+y_2-x_2,\\&y_1-x_1+x_2-y_2,y_1-x_1+y_2-x_2\}\end{aligned} ∣x1​−x2​∣+∣y1​−y2​∣=max{​x1​−y1​+x2​−y2​,x1​−y1​+y2​−x2​,y1​−x1​+x2​−y2​,y1​−x1​+y2​−x2​}​

然后把 xix_ixi​ 放到一起，yiy_iyi​ 放到一起。例如：

x1−y1+x2−y2=(+x1+x2)+(−y1−y2)x_1-y_1+x_2-y_2=(+x_1+x_2)+(-y_1-y_2) x1​−y1​+x2​−y2​=(+x1​+x2​)+(−y1​−y2​)

x1−y1+y2−x2=(+x1−x2)+(−y1+y2)x_1-y_1+y_2-x_2=(+x_1-x_2)+(-y_1+y_2) x1​−y1​+y2​−x2​=(+x1​−x2​)+(−y1​+y2​)

然后每个点 (x1,⋯ ,xk)(x_1,\cdots,x_k)(x1​,⋯,xk​) （代码为了方便，下标从 000 开始）可以表示成

∑i=1ktixi,ti∈{−1,1}\sum_{i=1}^k t_i x_i,t_i\in\{-1,1\} i=1∑k​ti​xi​,ti​∈{−1,1}

这个可以状压一下。

设某两不同点的某两个状态为 i,ji,ji,j，则若 iand⁡j=0,ior⁡j=2k−1i\operatorname{and}j=0,i\operatorname{or}j=2^k-1iandj=0,iorj=2k−1，则这是一个合法的曼哈顿距离，可以理解一下。

以上为套路。

每个线段树结点维护当前区间 2k2^k2k 个状态（代码中的数组 fff）中最优的那个值。

查询的答案为 max⁡i=02k−1{fi+f2k−i−1}\displaystyle\max_{i=0}^{2^{k-1}}\{f_i+f_{2^k-i-1}\}i=0max2k−1​{fi​+f2k−i−1​}。

时间复杂度 O(k2k⋅nlog⁡n)O(k2^k\cdot n\log n)O(k2k⋅nlogn)。

CF1190D Tokitsukaze and Strange Rectangle

https://www.luogu.com.cn/problem/CF1190D

Solution

线段树。

首先排个序是必要的，我选择了以 yyy 为第一关键字（必须的），以 xxx 为第二关键字 从大到小 排。

考虑第 iii 个点会对答案产生多少贡献。

显然，第 iii 个点会产生 ansl×ansransl\times ansransl×ansr 的贡献，其中，

  • anslanslansl 为横坐标位于 (−∞,xi](-\infty,x_i](−∞,xi​] 的点的个数。
  • ansransransr 为横坐标位于 [xi,+∞)[x_i,+\infty)[xi​,+∞) 的点的个数。

那么根据乘法原理即可证明。

但是有一种情况要特殊考虑，那就是纵坐标相同的时候。

当 yi=yi+1y_i=y_{i+1}yi​=yi+1​ 时，anslanslansl 要变成前 iii 个点落在区间 (xi+1,xi](x_{i+1},x_i](xi+1​,xi​] 的不同横坐标个数，这样才能保证不包含任何编号大于 iii 的点。

然后就可以线段树了，需要离散化。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF1175E Minimal Segment Cover

https://www.luogu.com.cn/problem/CF1175E

Solution

贪心 + 倍增。

贪心策略：记录一个数组 aaa，aia_iai​ 表示左端点在 0∼i0\sim i0∼i 的线段中，最右边的右端点。

然后每次询问就这样跳：

l→al→aal→aaal→⋯l\to a_l\to a_{a_l}\to a_{a_{a_l}}\to \cdots l→al​→aal​​→aaal​​​→⋯

一直跳到 ≥r\ge r≥r 为止。

若最终跳不到 rrr，答案为 −1-1−1。

否则答案为跳的次数加一（就是经过了这么多条线段，可以理解一下）。

这个可以倍增优化。

千万千万注意，aaa 数组和倍增数组的大小为值域！！否则你会悲惨地 RE on #10。

不用离散化真良心（

时间复杂度 O(mxlog⁡mx)O(mx\log mx)O(mxlogmx)，mxmxmx 为值域。

CF500E New Year Domino

https://www.luogu.com.cn/problem/CF500E

Solution

线段树。

General solution

每次询问的答案即把编号 [l,r][l,r][l,r] 的骨牌全推倒后，未被覆盖的区间长度之和。

这个可以离散化后用线段树维护。

然而我们仍然发现，左边的骨牌不该对右边的区间询问造成影响。

我们把询问离线下来，左端点从大到小排序，逐个添加骨牌并处理询问即可。

鉴于本题主函数细节较多，我在这里就详细讲一下吧qwq

Details

我们开一个 2n2n2n 的结构体数组记录骨牌。每个结构体记录 333 个值：

  1. 骨牌位置（ai.la_i.lai​.l）；
  2. 离散化后位置（ai.ra_i.rai​.r）；
  3. 编号（ai.ida_i.idai​.id）。

a1∼ana_1\sim a_na1​∼an​ 记录骨牌原来的信息，an+1∼a2na_{n+1}\sim a_{2n}an+1​∼a2n​ 记录骨牌倒下后右端点的信息。

先读入 ai.l(1≤i≤n)a_i.l(1\le i\le n)ai​.l(1≤i≤n)，通过骨牌长度计算出 ai+n.l(1≤i≤n)a_{i+n}.l(1\le i\le n)ai+n​.l(1≤i≤n)。

然后把 aaa 按 ai.la_i.lai​.l 从小到大排序，然后手动离散化。

在手动离散化的同时给 sizsizsiz 数组赋值，sizi=ai+1.l−ai.lsiz_i=a_{i+1}.l-a_i.lsizi​=ai+1​.l−ai​.l，即相邻两块骨牌间的距离。可知任何时候 [ai.l,ai+1.l)[a_{i}.l,a_{i+1}.l)[ai​.l,ai+1​.l) 这一线段的状态相同（状态指有无骨牌覆盖）。我们要维护这些状态，于是我们的线段树就是基于 sizsizsiz 数组而建的。

此时记离散化后值域 mx=a2n.rmx=a_{2n}.rmx=a2n​.r。

然后把 aaa 按编号从小到大排序，方便之后的处理。

  2. 询问右端点（bi.rb_i.rbi​.r）；

这部分的代码比较简单，就不在这里说了。

接下来就是处理询问了。

线段树结点内记录两个值：v,tagv,tagv,tag，vvv 代表区间内 未 被骨牌覆盖的个数，tagtagtag 代表该区间是否被骨牌所覆盖。

我们开一个指针 ppp，初始指向 ana_nan​。

让 ppp 一直往前移动，直到有询问为止。这个过程要一直持续下去，直到 ppp 指向 a1a_1a1​ 为止。

其中 VVV 是结点 uuu 的子树的结点组成的集合，bxb_xbx​ 是 x→1x\to 1x→1 的二类边长度。
