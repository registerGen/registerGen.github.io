CF52C Circular RMQ

https://www.luogu.com.cn/problem/CF52C

Solution

线段树傻逼题。

首先强行把下标变成从 111 开始。

如果 l≤rl\le rl≤r，就是基本的线段树。

如果 l>rl>rl>r，则在 [l,n][l,n][l,n] 上搞一搞，再在 [1,r][1,r][1,r] 上搞一搞就行了。

唯一有点 tricky 的地方就是判断是哪个操作（

在输进去 222 个数后，我们 getchar 一下，如果是空格，就是修改，否则就是 RMQ。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF413E Maze 2D

https://www.luogu.com.cn/problem/CF413E

翻译是我给的（逃

Solution

线段树。

这题 的简化版。

线段树内维护四个值：每段区间左上到右上，左上到右下，左下到右上，左下到右下的最短路：

struct Node
{
	int uu,ud,du,dd;
};

画个图就能得出 pushUp：

#define CN const Node

inline Node pushUp(CN& x,CN& y)
{
	Node res;
	res.uu=std::min(inf,std::min(x.uu+y.uu,x.ud+y.du)+1);
	res.ud=std::min(inf,std::min(x.uu+y.ud,x.ud+y.dd)+1);
	res.du=std::min(inf,std::min(x.dd+y.du,x.du+y.uu)+1);
	res.dd=std::min(inf,std::min(x.dd+y.dd,x.du+y.ud)+1);
	return res;
}

查询的时候分类讨论判一判就行了。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF1045G AI robots

https://www.luogu.com.cn/problem/CF1045G

Solution

CDQ 分治。

首先按视野从小到大排序，这样只要右边能看见左边就可以保证互相看见。

CDQ 内按智商排序，因为 kkk 固定，所以可以用莫队，莫队内套树状数组。

注意要离散化。

时间复杂度 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

CF165D Beard Graph

https://www.luogu.com.cn/problem/CF165D

Solution

树剖。

令白边边权为 111，黑边边权为 000。

线段树维护边权和。

修改就是线段树基本操作。

查询时，若边权不为 000，则输出 -1，否则就是 uuu 和 vvv 之间的边的条数。利用 depdepdep 数组和 LCA 计算即可。

时间复杂度 O(nlog⁡2n)O(n\log^2n)O(nlog2n)。

CF474F Ant colony

https://www.luogu.com.cn/problem/CF474F

Solution

线段树。

稍微分析一下可知，本题就是求 [l,r][l,r][l,r] 中 不等于 gcd⁡i=lrai\displaystyle \gcd_{i=l}^r a_ii=lgcdr​ai​ 的数的个数。

区间 gcd⁡\gcdgcd 和区间等于 gcd⁡i=lrai\displaystyle \gcd_{i=l}^r a_ii=lgcdr​ai​ 个数都有可加性，直接线段树就好了。

inline void pushUp(int i)
{
	t[i].gcd=gcd(t[ls(i)].gcd,t[rs(i)].gcd);
	t[i].sum=(t[i].gcd==t[ls(i)].gcd?t[ls(i)].sum:0)+(t[i].gcd==t[rs(i)].gcd?t[rs(i)].sum:0);
}

时间复杂度 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

CF689D Friends and Subsequences

https://www.luogu.com.cn/problem/CF689D

Solution

二分 + ST 表。

考虑当 lll 固定时，如何快速计算对答案的贡献。

我们发现，当 lll 固定时，f(r)=max⁡i=lraif(r)=\displaystyle\max_{i=l}^r a_if(r)=i=lmaxr​ai​ 单调不减，g(r)=min⁡i=lraig(r)=\displaystyle\min_{i=l}^r a_ig(r)=i=lminr​ai​ 单调不增。

蓝色段的长度就是对答案的贡献。

我们可以二分求出蓝色段的左端点和右端点，求 min⁡\minmin，max⁡\maxmax 时用 ST 表，因为不带修。时间复杂度 O(log⁡n)O(\log n)O(logn)。

总时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

CF474E Pillars

https://www.luogu.com.cn/problem/CF474E

Solution

DS 优化 DP。

这题 DP 很好想，跟 LIS 差不多。

设 fif_ifi​ 代表取到 iii 且 iii 必须取的答案。

则：

fi=max⁡{fj}+[∣ai−aj∣≥d]f_i=\max\{f_j\}+[|a_i-a_j|\ge d] fi​=max{fj​}+[∣ai​−aj​∣≥d]

即：

fi=max⁡{fj}+[aj≤ai−d or aj≥ai+d]f_i=\max\{f_j\}+[a_j\le a_i-d \text{ or } a_j\ge a_i+d] fi​=max{fj​}+[aj​≤ai​−d or aj​≥ai​+d]

显然是 线段树/树状数组 优化 DP，注意要离散化后二分位置。
